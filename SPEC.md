# SPEC — アルゴリズムゲーム 仕様書

## 概要

グリッド上のキャラクターをゴールまで導くことでアルゴリズム的思考を学ぶ学習ゲーム。
難易度に応じて2つのプログラミング方式を提供する。

- **初級（ブロックモード）**: Scratch 風のブロックプログラミング。命令ブロックを並べてプログラムを組む。
- **上級（コードモード）**: C言語で本物のプログラムを書き、同じ課題を解く。

## ゲームルール

### 共通

- ステージは N×M のグリッド。マスの種類: 床 / 壁 / スタート / ゴール / （拡張候補: スイッチ、鍵、穴）。
- キャラクターはスタート地点に向き付きで配置される。
- プログラムを実行し、キャラクターがゴールに到達すればクリア。
- 壁への衝突・盤外への移動は失敗（その場で停止し、失敗表示）。
- 実行ステップ数の上限あり（無限ループ対策）。
- クリア時に星1〜3で評価。初級は使用ブロック数、上級は実行ステップ数（move/turn の回数）で判定する。
- ステージ数: 初級200問 + 上級100問。`front/scripts/gen-stages.ts` が生成し、
  全問についてクリア可能性を機械検証している（`data/stages.json`）。

### 初級: ブロックモード

利用可能なブロック（ステージごとに使用可能セットを制限できる）:

| ブロック | 意味 |
|---|---|
| まえにすすむ | 向いている方向に1マス移動 |
| ひだりをむく | 左に90度回転 |
| みぎをむく | 右に90度回転 |
| くりかえす (N回) | 子ブロック列をN回繰り返す |
| ずっとくりかえす | ゴールするまで繰り返す（ステップ上限で必ず停止） |
| もし まえがかべ なら | 前方が壁のときだけ子ブロック列を実行する |

- ブロックはパレットからワークスペースへ追加・削除・並び替えできる（クリックとドラッグ&ドロップの両対応）。
- 実行はトレースを1イベントずつアニメーション再生する（速度調整つき）。
- 効果音・BGM あり（WebAudio 合成。トグルで ON/OFF）。

### 上級: コードモード

- ブラウザ上のエディタで C 言語のコードを書く。
- 問題は2系統(合計100問。同一プログラムの使い回しで全問クリアできないようにするため):
  - **ナビゲーション問題(60問)**: 迷路や地形をゴールまで移動する。
    星は実行ステップ数(move/turn の回数)で評価。
  - **配列問題(40問)**: 値の列を swap 操作で目標の並びにする。
    課題は5種類: 昇順ソート / 降順ソート / 逆順 / 最大値を最後へ / 最小値を先頭へ。
    星は交換(swap)回数で評価(例: 昇順ソートはバブルソートで★2、
    選択ソートなど交換回数の少ない解法で★3)。
- 提供 API（ヘッダ `game.h`）。その問題で使えない関数を呼ぶと失敗になる:
  ```c
  // ナビゲーション問題
  void move_forward(void);  // 1マス前進
  void turn_left(void);     // 左回転
  void turn_right(void);    // 右回転
  int  is_wall_ahead(void); // 前方が壁なら1
  int  is_goal(void);       // ゴール上なら1

  // 配列問題
  int  array_length(void);      // 配列の長さ
  int  get_value(int i);        // i番目の値(範囲外は失敗)
  void swap_values(int i, int j); // i番目とj番目を交換。条件達成の瞬間にクリア
  ```
- ユーザーコードはバックエンド（Rust）でサンドボックスコンパイル・実行し、
  移動コマンド列（トレース）をフロントへ返して再生する。
- コンパイルエラー・実行時エラーはエディタ下部に表示。
- センサー API（is_wall_ahead / is_goal）の呼び出しもステップとして数え、
  ステップ上限・CPU 時間制限・実行タイムアウトの三段で無限ループを止める。
- エディタは CodeMirror（C シンタックスハイライト付き）。

## 画面構成

1. **タイトル画面**: モード選択（初級 / 上級）。
2. **ステージ選択画面**: ステージ一覧、クリア状況・星表示。
3. **プレイ画面**:
   - 左: グリッド盤面（キャラクター、ゴール表示、実行アニメーション）
   - 右: ブロックワークスペース（初級） / コードエディタ（上級）
   - 下: 実行 / 停止 / リセット、実行速度調整
4. **クリアダイアログ**: 星評価、次のステージへ。

## 技術構成

| 層 | 技術 |
|---|---|
| フロントエンド | TypeScript + React + Vite |
| バックエンド | Rust + axum。ステージ配信、Cコード実行サンドボックス、進捗保存（SQLite） |
| 通信 | REST + JSON（実行トレース） |

### API（バックエンド）

- `GET /api/stages` — ステージ一覧
- `GET /api/stages/:id` — ステージ詳細
- `POST /api/run/c` — Cコードを受け取り、コンパイル・実行してトレースを返す
- `GET /api/progress` — クリア状況取得
- `POST /api/progress` — クリア状況保存（ベスト星数のみ更新）

## 実装状況

- 初級ブロックモード: ブラウザ内インタプリタで完全動作。
- 上級コードモード: バックエンドの C サンドボックスで実行し、トレースを再生。
- ステージデータはバックエンド API から取得し、API 停止時はフロント同梱データに
  フォールバックする（バックエンドなしでも初級は完全に遊べる）。
- 進捗は localStorage と SQLite（API 経由）の両方に保存し、良い方をマージする。
